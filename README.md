	<p class="tit">ВВЕДЕНИЕ</p>
	<p class="par">Всемирная компьютерная сеть Интернет — это глобальная система взаимосвязанных компьютерных сетей, которая позволяет пользователям по всему миру обмениваться информацией и взаимодействовать друг с другом через различные устройства.
Принципы построения и функционирования Всемирной компьютерной сети Интернет основаны на нескольких ключевых аспектах, обеспечивающих ее масштабируемость, надежность и доступность. Вот основные из них:
</p>
	<p class="par">1. Децентрализация<br>
Интернет не управляется единым центром или организацией. Вместо этого он представляет собой совокупность независимых сетей, соединенных между собой. Это обеспечивает устойчивость к сбоям и атакам, так как отказ одной части сети не приводит к остановке всей системы.
</p>
	<p class="par">2. Открытые стандарты<br>
Все технологии и протоколы, используемые в Интернете, разрабатываются и поддерживаются международными организациями, такими как IETF (Internet Engineering Task Force). Эти стандарты открыты и доступны всем участникам сети, что способствует совместимости различных устройств и программных решений.
</p>
	<p class="par">3. Клиент-серверная архитектура<br>
В основе взаимодействия в Интернете лежит модель клиент-сервер. Серверы предоставляют услуги (например, веб-сайты, файлы, базы данных), а клиенты запрашивают эти услуги. Такая архитектура позволяет эффективно распределять нагрузку и обеспечивать масштабирование.
</p>
	<p class="par">4. Маршрутизация пакетов<br>
Данные передаются по сети в виде небольших блоков, называемых пакетами. Каждый пакет содержит адрес отправителя и получателя, а также часть передаваемой информации. Маршрутизаторы направляют пакеты по оптимальному маршруту до конечного пункта назначения.
</p>
	<p class="par">5. Многоуровневая структура протоколов<br>
Для обеспечения надежной передачи данных используются несколько уровней протоколов. Например, TCP (Transmission Control Protocol) отвечает за надежную доставку данных, а IP (Internet Protocol) — за их маршрутизацию. Протокол HTTP используется для передачи гипертекста (веб-страниц).
</p>
	<p class="par">6. DNS (Domain Name System)<br>
Система доменных имен преобразует понятные человеку доменные имена (например, example.com) в IP-адреса, которые используются для связи между устройствами в сети. DNS-сервера распределяют эту задачу по иерархической структуре, обеспечивая быструю и эффективную работу.
</p>
	<p class="par">7. IP-адресация<br>
Каждому устройству, подключенному к Интернету, присваивается уникальный IP-адрес, который служит его идентификатором в сети. Существует два основных типа IP-адресов: IPv4 (32 бита) и IPv6 (128 бит), последний из которых был введен для решения проблемы нехватки адресного пространства.
</p>
	<p class="par">8. Кэширование и CDN (Content Delivery Network)<br>
Чтобы уменьшить задержки при передаче данных и снизить нагрузку на серверы, часто используется кэширование контента на промежуточных серверах и CDN. Это позволяет доставлять контент быстрее и ближе к конечному пользователю.
</p>
	<p class="par">9. Безопасность<br>
Безопасность является важным аспектом функционирования Интернета. Для защиты данных используются такие методы, как шифрование (SSL/TLS), аутентификация пользователей, файрволлы и антивирусное ПО. Кроме того, существуют специальные протоколы для безопасной передачи данных, например, HTTPS.
</p>
	<p class="par">10. Масштабируемость<br>
Интернет спроектирован таким образом, чтобы поддерживать постоянное увеличение числа пользователей и объема передаваемых данных. Это достигается благодаря использованию динамической маршрутизации, многоуровневой архитектуры и возможности добавления новых узлов без необходимости изменения существующих компонентов.
</p>
	<p class="par">Эти принципы позволяют Всемирной компьютерной сети Интернет оставаться гибкой, устойчивой и доступной для миллионов пользователей по всему миру.</p>
	<p class="par">В качестве объекта исследования выступает литература по теме магистерской диссертиции. Предметом исследования выступают способы поиска научных публикаций в сети Интернет.</p>
	<p class="par">Цель написания реферативной работы — получить более глубокую теоретическую и практическую подготовку по специальности на основе самостоятельного изучения и обобщения научной, учебной литературы и нормативных документов, а также формирования умений и навыков по работе в сети.</p>
	<p class="par">Задачи:<br>
1. ознакомиться с всемирной компьютерной сетью Интернет и используемыми в ней технологиями;<br>
2. освоить приемы эффективного поиска необходимой информации в сетевых ресурсах;<br>
3. получить практический опыт поиска информации по заданной теме;<br>
4. освоить методы создания Web-документов;<br>
5. получить практический опыт создания Web-страниц на основе найденных в сети документов;<br>
6. сделать обзор литературы по теме диссертации, используя ресурсы сети Интернет.
</p>
